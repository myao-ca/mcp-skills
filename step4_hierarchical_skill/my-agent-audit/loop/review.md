# 死循环风险审查标准

## 审查要点

### 轮次限制
- agentic loop 是否有 max_turns 上限？
- 子 agent 是否有独立的、更严格的轮次限制？
- 达到上限后是否有清晰的退出和告知机制？

### 重复检测
- 是否检测 LLM 反复调用同一个 tool 传同样参数（原地踏步）？
- 是否检测连续多轮没有实质进展的情况？
- 重复检测触发后有没有降级策略（告知用户/换策略）？

### 超时机制
- 单次 tool 调用是否有超时限制？
- 并行 tool 调用是否有 timeout 参数（避免某个 tool 卡死整个流程）？
- 整个 agentic session 是否有总体时间上限？

### 错误重试
- 重试逻辑是否有上限（避免无限重试）？
- 重试间隔是否有退避策略（避免瞬间打爆 API）？
- 连续失败后是否有放弃机制？
