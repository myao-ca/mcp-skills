# 安全边界审查标准

## 审查要点

### 危险操作拦截
- 是否有删除文件、执行任意代码、写入系统文件等危险操作的拦截？
- 拦截是否在执行前（而不是执行后）？
- 是否有明确的白名单/黑名单机制？

### 路径安全
- 文件操作是否有 workspace 边界限制？
- 是否用 realpath() 防止路径穿越攻击（../../ 等）？
- 未配置 WORKSPACE_PATH 时是否默认拒绝（fail-safe）？

### 权限最小化
- 每个 tool 的能力范围是否最小化？
- 有没有本该只读但却支持写入的 tool？
- 对外暴露的接口是否超出实际需要？

### 用户验证
- 是否有用户身份绑定（如 chat_id 白名单）？
- 多用户场景下数据是否完全隔离？
- 是否有防止未授权访问的机制？
