# 上下文管理审查标准

## 审查要点

### Token 溢出防护
- 是否有 token 或消息数量的上限检测？
- 接近上限时是否有压缩或截断策略？
- 压缩策略是否保留了关键信息（tool 调用结果、重要决策）？

### 历史保留策略
- 压缩时是否处理了"孤立 tool_result"问题（对应的 tool_use 被删掉会导致 API 报错）？
- 保留最近 N 条消息的策略是否合理？
- 是否区分了重要消息和冗余消息？

### 压缩成本
- 压缩是否使用了同等级别的模型（成本是否过高）？
- 是否可以用更小的模型做摘要？
- 压缩频率是否合理（太频繁 = 成本高，太少 = 溢出风险）？

### 子 Agent 隔离
- 子 agent 的 context 是否和主 agent 隔离？
- 子 agent 完成后是否只把结果返回主 agent（而非整个 context）？
- ephemeral session 机制是否正确实现？
