# Tool 设计审查标准

## 审查要点

### Description 质量
- description 是否清晰说明了"什么时候用"而不只是"是什么"？
- LLM 能否通过 description 准确判断何时调用？
- 有没有容易混淆的 tool（功能重叠，LLM 难以选择）？

### 参数设计
- 参数是否最小化？有没有不必要的参数？
- 必填 / 选填划分是否合理？
- 参数名和描述是否足够自解释？

### 返回值
- 返回值格式是否稳定？同一个 tool 在不同情况下格式一致吗？
- 错误时的返回值是否有助于 LLM 自我修正？
- 返回值是否过于冗长（浪费 token）？

### 粒度
- 有没有过于宽泛的 tool（一个 tool 做太多事）？
- 有没有过于细碎的 tool（应该合并的拆得太散）？
